Single Page App "PublicAppsHub"

Esta web es el proyecto de Arquitectura de Software 2016 - UPNA. Implementado como SPA usando la librería Backbone. A continuación se indica el enunciado del proyecto

Implementa una App para Android o una Single Page App “PublicAppsHub” con la siguiente funcionalidad:

A) la App “PublicAppsHub” tendrá una pantalla inicial donde estarán todas las Apps publicadas ordenadas por categorias. Pantalla 1.

B) cada App que se ha publicado se verá en una vista mostrando la información que escribió el usuario que la público. Pantalla 2.

C) los usuarios de “PublicAppsHub”  pueden publicar el diseño de Apps completando la siguiente información: nombre de la app, categoria, descripción textual, y descripción visual (que consiste en una conjunto de imágenes). Pantalla 3.

Las categorias existentes son Apps Educativas, Salud, Transporte.

_____________________________________________________________________________

En la segunda fase del proyecto (con entrega en segunda semana de Abril) se añadirá la siguiente funcionalidad:

D) Para cada app pública se podrá añadir implementaciones del código: para cada código se puede añadir el nombre de implementación, autor de implementación, localización de un repositorio con el código fuente, y plataformas en las que esta implementada. Pantalla 4a y 4b

E) Para cada implementación se podrá añadir despligues del código: para cada despliegue se puede añadir el nombre del despliegue, la URL de acceso, el area geográfica de sus usuario y el nombre de la empresa que gestiona el servicio. Pantallas 5a y 5b.

_____________________________________________________________________________

Se entregará:

-el código de la implementación: un zip de la carpeta de Eclipse o Netbeans donde se abre el proyecto,

-una visualización de la app funcionando: un video en formato MP4 con duración menor a 20 segundos,

-un documento en formato PDF explicando el diseño MVC que corresponde con la implementación.

Criterios de evaluación:

 5 puntos: todas las pantallas implementadas y funcionando correctamente. (si no se cumple este criterio no se puntuarán el criterio de documentación).

+2 puntos: si se implementa como SPA en backbone.js.

+3 puntos: documentación explicando el diseño MVC, el diseño deberá separar el código según el patrón y corresponder con la implementación.

(Pueden darse hasta 2 puntos más si se implementa alguna funcionalidad extra, o se resuelve de manera sobresaliente).
